# 評価依頼書: 既存物体検出モデルの導入可否評価

> 本ドキュメントは PdM 視点で書かれた評価依頼書である。本プロジェクトでは物体検出モデルを開発せず、既存の学習済みモデルを調達して利用する方針であり、本書はその導入可否を QA に評価依頼するための要件を定義する。受け入れ条件 (AC-1〜AC-5) は[テスト計画](./test-plan.md)・[テスト設計](./test-design.md)・[テスト完了レポート](./test-report.md)を通じて参照される。

## 1. 背景・課題

当社は、屋内外の通路・構内における安全補助を目的に、カメラ映像から歩行者および車両を検出し、現場担当者へ注意喚起する補助機能の導入を検討している。

検出モデルそのものは自社開発しない。物体検出は既に成熟した技術であり、高品質な学習済みモデルが公開されているため、これを調達して利用する方が合理的である。モデルの選定・改良はエンジニアチームの判断領域とする。

一方で、「**既存モデルを、当社のこの用途で本番投入してよいか**」という品質判断は、モデルの精度指標を眺めるだけでは下せない。当社の用途に固有の品質要件（とりわけ「見逃しをどこまで許容できないか」）に照らして、モデルの出力を外部から評価する必要がある。本書は、その評価を QA に依頼するものである。

## 2. ユーザーストーリー

- **現場担当者として**、人や車の見落とし（検知漏れ）だけは避けてほしい。なぜなら、検知漏れは安全上の事故に直結しうるから。
- **現場担当者として**、多少の誤検知は受け入れる用意があるが、頻繁すぎる誤作動は困る。なぜなら、不要な介入が業務を妨げ、やがて警告そのものを信用しなくなってしまうから。
- **PdM として**、調達候補のモデルが当社の用途の品質要件を満たすか、本番投入の前に客観的な根拠をもって判断したい。なぜなら、安全に関わる機能を、性能の裏付けなく導入することはできないから。

## 3. 評価対象 (SUT) の与件

本評価が前提とする、調達候補モデルの仕様を以下に記す。これらは所与の事実であり、評価によって変更する対象ではない。

**入力:**
- 静止画像（カメラから取得したフレームを想定）

**出力:**
- 検出された各物体について、バウンディングボックス（位置・大きさ）、クラス名、信頼度スコア

**対象クラス:**
- 本用途で関心があるのは person（歩行者）および車両系クラス（car, bicycle, bus, truck 等）。COCO の検出クラスに含まれる範囲とする。
- 上記以外のクラスの検出結果は、本用途では評価対象としない。

**前提・制約:**
- モデルは COCO データセットで学習済みであり、当社の現場映像で追加学習（ファインチューニング）はしていない。
- 本評価ではモデルの再学習・改良は行わない。出力のみを評価する。

## 4. 受け入れ条件（Acceptance Criteria）

借り物のモデルを評価するため、各 AC は「機能の有無」ではなく「当社用途における品質水準」として定義する。各 AC を計測可能な指標へ翻訳することと、具体的な合格水準の設定は、本書の品質要件（§5）に基づいて QA がテスト計画で起案し、PdM が承認する。

### AC-1: 対象物の検出
- 本用途の対象クラス（歩行者・車両）について、評価データ中に実在する対象を、見逃さず検出できること。
- 「見逃し（実在するのに検出しない）」を最重要の品質課題とみなす。

### AC-2: 検出位置の妥当性
- 検出されたバウンディングボックスが、対象の実際の位置と十分に重なっていること。位置のずれが大きい検出は、現場での注意喚起の信頼性を損なうため、許容される位置精度を別途定める。

### AC-3: 誤検出の許容範囲
- 誤検出（実在しない対象を検出する）が、現場運用上許容できる範囲に収まっていること。
- 誤検出は見逃しより重大度は低いものの、本用途では過剰反応（不要な介入）という二次的な実害を生むため（§5 参照）、許容上限を設け、これを超えないことを必須の合格条件とする。

### AC-4: 条件別の性能劣化
- 小さく写る対象、遮蔽のある対象、低照度・混雑など、現場で想定される条件下での性能劣化が、許容範囲に収まっていること。

### AC-5: 推論レスポンス
- 1 フレームあたりの推論時間が、用途上の要件（リアルタイム性）を満たすこと。

## 5. 未決事項・懸念事項

### 未決事項
- [ ] 見逃しの合格水準（実在対象のうち、どの程度を取りこぼさず検出できれば合格とするか）
- [ ] 誤検出の許容上限（検出結果のうち、誤りがどの程度までなら現場運用上許容できるか）
- [ ] 検出位置の許容ずれ（位置精度の合格水準）
- [ ] 評価用データのサブセット（通常・小物体・低照度・混雑）をどう構成するか
- [ ] 用途を本書の単一用途に絞るか、対比用途も評価範囲に含めるか

### 懸念事項
- 調達候補モデルは COCO 学習済みであり、当社の現場映像とはドメインが異なる。COCO 上の公称性能が、当社用途でそのまま再現される保証はない（だからこそ本評価を行う）。
- 評価に用いる公開データセットのシーンが、当社の現場条件をどこまで代表しているか。代表性の限界は評価結果の解釈に明記する必要がある。

## 6. 技術構成（概要）

- 評価対象 (SUT): YOLO 系学習済みモデル（ローカル推論。外部 API 課金は発生しない）
- 評価データ: COCO / VOC 等の公開データセット。**正解アノテーションが客観的に与えられている**点が本評価の前提を支える
- 評価基盤の具体（使用する評価指標・ライブラリ・閾値設計など）はテスト計画で QA が起案する

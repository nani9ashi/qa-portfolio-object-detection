# テスト設計書: 既存物体検出モデルの導入可否評価

## 1. 概要

本設計書は、テスト計画書（[test-plan.md](./test-plan.md)）で定義したテスト戦略に基づき、具体的な評価観点と評価ケースを記述する。

### 1.1 対象範囲

- YOLO 系学習済みモデルの検出出力に対する評価ケース
- 対応する AC: AC-1 〜 AC-5（AC-4 は AC-4a / AC-4b に分割）

### 1.2 テスト設計の方針

- 評価データの構成は、サブセットの代表性と作成の現実性を踏まえて定義する
- 各評価ケースには優先度（高/中/低）を付与する（判定基準は[テスト計画書](./test-plan.md) §4.5 参照）
- 各評価ケースには観点 ID と対応 AC を明示し、トレーサビリティを確保する
- 条件別評価は再現率と適合率を独立した 2 軸で扱い、両方を合格条件とする（[test-plan](./test-plan.md) §2.4, §5.2 参照）
- 検出性能を単一スコアに統合する指標（F1 等）は採用しない（[test-plan](./test-plan.md) §2.4 参照）

## 2. 評価観点一覧

評価観点は以下のとおり定義する。各観点はセクション 3 の評価ケース表から参照される。

| 観点 ID | 観点 | カテゴリ | 優先度 |
|---|---|---|---|
| V-01 | 全体の再現率（対象クラス全体） | 全体性能 | 高 |
| V-02 | 全体の適合率（対象クラス全体） | 全体性能 | 中 |
| V-03 | 平均 IoU（真陽性検出） | 全体性能 | 中 |
| V-04 | mAP の算出（参考） | 全体性能 | 低 |
| V-05 | クラス別の再現率・適合率（参考） | 全体性能 | 低 |
| V-06 | 物体サイズ別の再現率 | 条件別評価 | 高 |
| V-07 | 物体サイズ別の適合率 | 条件別評価 | 中 |
| V-08 | 混雑度別の再現率 | 条件別評価 | 高 |
| V-09 | 混雑度別の適合率 | 条件別評価 | 中 |
| V-10 | 明度別の再現率（サブセット成立時のみ） | 条件別評価 | 高 |
| V-11 | 明度別の適合率（サブセット成立時のみ） | 条件別評価 | 中 |

優先度の判定基準（[test-plan](./test-plan.md) §4.5）に従い、安全に直結する観点（再現率、条件別の見逃し）を高、運用品質に関わる観点（適合率、IoU、推論時間）を中、補助・参考的な観点（mAP、クラス別内訳）を低とした。

## 3. 評価ケース一覧

評価ケースは観点を計測可能な作業単位に落としたもの。条件別評価については、再現率と適合率の両方を 1 ケース内で独立に算出する設計とする（再現率と適合率を別ケースに分けない理由は、同じサブセット・同じ運用点で算出するため、評価実行としては一体だからである）。

| ID | 優先度 | 対応 AC | 観点 | 評価内容 | 合格基準 |
|---|---|---|---|---|---|
| EVAL-001 | 高 | AC-1 / AC-3 / AC-2 | V-01 / V-02 / V-03 | 評価対象画像全体に対する検出結果と正解アノテーションを突き合わせ、対象クラス全体の再現率・適合率・平均 IoU（真陽性検出）を算出する。IoU 閾値 0.5、運用点 conf 0.25。 | 再現率 ≥ 95% かつ 適合率 ≥ 70% かつ 平均 IoU ≥ 0.7（AND 判定） |
| EVAL-002 | 低 | AC-1 / AC-2 | V-04 | 評価対象画像全体について、対象クラスの mAP@0.5 と mAP@0.5:0.95 を算出する（IoU を厳格化した平均）。 | 合否対象外。参考値として記録 |
| EVAL-003 | 低 | AC-1 / AC-3 | V-05 | 対象クラスごと（person / bicycle / car / bus / truck）の再現率・適合率を算出する。クラス間の偏りを観察する。 | 合否対象外。参考値として記録。特定クラスで極端な低下があれば §6 で考察 |
| EVAL-004 | 高 | AC-4a / AC-4b | V-06 / V-07 | 物体サイズ別（small / medium / large、COCO 標準定義）に再現率と適合率を独立に算出する。サブセットは画像単位ではなく対象オブジェクト単位で分類。 | 各サイズで再現率 ≥ 80% かつ 適合率 ≥ 70%（AND 判定） |
| EVAL-005 | 高 | AC-4a / AC-4b | V-08 / V-09 | 混雑度別（画像内の対象クラス数による分類：低 = 0-2 個、中 = 3-5 個、高 = 6 個以上）に再現率と適合率を独立に算出する。 | 各混雑度で再現率 ≥ 80% かつ 適合率 ≥ 70%（AND 判定） |
| EVAL-006 | 高 | AC-4a / AC-4b | V-10 / V-11 | 明度別（画像のグレースケール平均輝度による分類）に再現率と適合率を独立に算出する。サブセット成立条件は §4.3 参照。 | サブセットが成立した場合: 各明度で再現率 ≥ 80% かつ 適合率 ≥ 70%。サブセットが成立しない場合: 本ケースは不実施とし、その旨を §6 で記録 |
| EVAL-007 | 中 | AC-1 / AC-3 | V-12 | 信頼度スコア下限を 0.05 刻みで 0.05 〜 0.95 まで動かし、各点での再現率と適合率を算出して PR 曲線を描く。 | 合否対象外。判断材料として提示。EVAL-001 で不合格となった場合、運用点変更で合格可能か検討する材料となる |
| EVAL-008 | 中 | AC-5 | V-13 | 1 フレームあたりの推論時間を計測する。GPU 環境で動作させ、ウォームアップ 10 フレームを除いた定常状態の平均時間を算出する。 | 平均推論時間 ≤ 100ms / フレーム |

評価ケース全体での合否は、[test-plan](./test-plan.md) §5.2 の AND 判定方針に従う。**EVAL-001 / EVAL-004 / EVAL-005 / EVAL-006（成立時）/ EVAL-008 のいずれか一つでも不合格があれば、本用途への導入は不可と判定する**。EVAL-002 / EVAL-003 / EVAL-007 は参考値であり、合否判定の直接的対象ではない。

### 3.1 参考値とした項目の根拠と限界

EVAL-002（mAP）、EVAL-003（クラス別）、EVAL-007（PR 曲線）を合否対象から外し参考値としたのは、以下の理由による。

- **EVAL-002（mAP）**: mAP は複数 IoU 閾値を平均した統合的指標で、本評価の合否判定が単一の運用点（IoU 0.5）で立てられていることと粒度が合わない。物体検出の慣行として参考値で報告するに留める。

- **EVAL-003（クラス別）**: 合格水準が「対象クラス全体」（AC-1 / AC-3）で立てられているため、クラス別はその内訳として参考値とする。これは現状の合格水準の立て方に従った帰結である。
  ただし、この扱いには**設計上の限界**がある。COCO val2017 では person のサンプル数が圧倒的に多く、bus・truck などは少ない。全体平均はクラス間の偏りを隠す可能性があり、例えば「person だけ高水準で他は低水準」というモデルでも全体としては合格になりうる。これは「歩行者と車両を検出する」という本機能の用途要件と必ずしも整合しない。本評価で参考値扱いとする判断は、現状の合格水準が全体で立てられていることに合わせた整理であり、本来はクラス別の合格水準を立てるべきという論点が残っている。本評価では EVAL-003 の結果を §6 で報告し、特定クラスで極端な低下があれば「次フェーズでクラス別合格水準を立てるべきか」を考察材料として提示する。

- **EVAL-007（PR 曲線）**: PR 曲線は信頼度閾値を動かしたときのトレードオフを示すもので、特定の運用点での合否判定とは別の用途を持つ。運用点変更で改善可能かを検討する判断材料として位置づける。

## 4. 評価対象データ

本評価で使用する評価データの構成と、サブセットの定義を以下に記す。

### 4.1 ベースデータセット

- **データセット**: COCO val2017
- **画像数**: 5000 枚（フル）を基本とする。実行時間が問題になる場合は段階的に縮小する（最小 1000 枚を下限とし、それ以下にする場合はテスト計画書に追記）
- **対象クラス**: person（COCO ID 0）、bicycle（ID 1）、car（ID 2）、bus（ID 5）、truck（ID 7）の 5 クラス
- **対象外クラス**: COCO の他クラス（motorcycle, train 等の輸送機器を含む）は本評価の対象とせず、検出されても無視する

### 4.2 運用点と IoU 閾値

- **運用点（信頼度スコア下限）**: 0.25 を初期値とする。これは ultralytics の YOLOv8 系のデフォルト値で、物体検出の慣行に従う
- **IoU 閾値（検出と正解の対応付け）**: 0.5 を主評価、0.75 を補助評価（EVAL-002 の mAP@0.5:0.95 に内包）

### 4.3 条件別サブセットの定義

**物体サイズ別（EVAL-004）**

COCO 標準の定義に従う。

| サイズ | 定義 |
|---|---|
| small | 面積 ≤ 32² = 1024 px² |
| medium | 1024 px² < 面積 ≤ 96² = 9216 px² |
| large | 9216 px² < 面積 |

分類は画像単位ではなく対象オブジェクト単位で行う（1 枚の画像に small と large の両方が含まれることがある）。pycocotools の標準機能で算出可能。

**混雑度別（EVAL-005）**

画像内の対象クラス数（person, bicycle, car, bus, truck の合計）で画像を分類する。

| 混雑度 | 定義 |
|---|---|
| 低 | 1 画像あたりの対象数 0-2 個 |
| 中 | 1 画像あたりの対象数 3-5 個 |
| 高 | 1 画像あたりの対象数 6 個以上 |

対象数 0 個の画像（背景のみ）も「低」に含める。これは「対象がない画像で誤検出を出さないこと」の評価に有意な集合となる。

**明度別（EVAL-006）**

画像のグレースケール平均輝度（0-255 の範囲）で画像を分類する。

| 明度 | 定義 |
|---|---|
| 低 | 平均輝度 < 80 |
| 中 | 80 ≤ 平均輝度 < 160 |
| 高 | 160 ≤ 平均輝度 |

**サブセット成立条件**: 「低」明度サブセットが 30 枚以上得られた場合のみ、本評価を実施する。30 枚未満の場合は、統計的な意味のある評価が困難なため、本ケースは不実施とし、その旨をテスト完了レポートで明示する。30 枚という閾値は、各サイズ / 混雑度サブセットでも同様に最小要件として適用する。

## 5. 評価実行の前提

### 5.1 評価環境

- **ハードウェア**: GPU 環境（本番運用相当）。CPU 環境での実行も可能だが、AC-5（推論時間）の評価は GPU 環境を必須とする
- **モデル**: YOLOv8n（推奨）。サイズの異なるモデルでの再評価は本評価のスコープ外
- **評価ライブラリ**: ultralytics、pycocotools、PyTorch

### 5.2 評価実行のシード性

YOLO の推論は決定的（同じ入力に対して同じ出力）であるため、評価結果は再現可能である。同じ評価データ・同じモデル・同じ運用点で複数回実行しても、検出結果と指標値は完全に一致する。よって、評価実行は 1 回で十分であり、同一入力に対する反復実行や複数試行の統計処理は本評価では不要である。

### 5.3 評価結果の記録

各評価ケース（EVAL-001 〜 EVAL-008）の実行結果を JSON / CSV 形式で記録する。記録内容には以下を含める。

- ケース ID と実行日時
- 算出された指標値（再現率、適合率、IoU、mAP、推論時間など）
- 使用した評価データの規模（画像数、対象数）と運用点
- 合否判定（合格 / 不合格 / 参考値）

これらの記録は、テスト完了レポートで結果を分析・報告する際の一次資料となる。

---

## 6. 評価結果の報告方針

評価実行後、テスト完了レポートで以下を報告する。

- **EVAL-001 〜 EVAL-008 の結果一覧**: 各ケースの算出値と合否
- **総合判定**: AND 判定方針に従った導入可否の判定
- **不合格となった項目の分析**: どの観点で合格水準を下回ったか、原因の考察
- **PR 曲線（EVAL-007）の解釈**: 運用点変更で改善可能かの検討材料
- **参考値（EVAL-002 / EVAL-003）の傾向**: mAP やクラス別の偏りから読み取れる特徴
- **不実施ケース**（EVAL-006 でサブセット不成立の場合）の明示と、その理由
- **次フェーズへの提言**: 不合格となった場合、Issue で議論された対応選択肢（モデル変更、信頼度閾値調整、用途限定）のうち、どれが有望かを評価結果に基づき提案する

## 補足: 評価ケース ID の付与方針

評価ケース ID は `EVAL-XXX` の形式とした。本評価が単一の検出評価に集約されているため、テスト層別の接頭辞による区別は設けていない。
